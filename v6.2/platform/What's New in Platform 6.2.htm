What's New in Platform 6.2

This section outlines a summary of the highly anticipated features updates, enhancements, and performance improvements bundled with the Infinite Blue Platform version 6.2. Customers currently running version 5.0 or later are eligible to upgrade to version 6.2. If you are running an earlier version, please contact support for additional information.

Here are the major updates:

Features/Enhancements

1. Job enhancements

  1. Job Priority:

    Administrators can now prioritize batch jobs to optimize job queuing and job processing. If multiple batch jobs are scheduled to run at the same time, this numerical priority will control the order in which batch jobs are run.

    For detailed information on this, see Batch Jobs.

  2. Job Cancellation:

    Prior to 6.2.0.0, job cancellations were possible only for master tenant administrators. Now, job monitoring and cancellation are possible at the tenant level for in-process or pending jobs like Import, Mass Run and Mass Update. This enhancement would allow the tenant users to monitor the jobs effectively.

    For detailed information on this, see Components and Monitoring Setup.

Portal User Password Management

The new Change Password portal link allows authenticated portal users to manage their own passwords directly in the portal. Administrators can add this component to any portal page in the Page Designer.

For detailed information on this, see Change Password for Portal Users.

Graal JavaScript engine

Prior to 6.2.0.0, the Platform used the Rhino (Mozilla) JavaScript engine to run server-side JavaScript. Tenant administrators can now choose to run server-side JavaScript with the Graal JS (by Graal VM) engine to enhance performance. Platform developers can also now utilize JavaScript features like promises and modules after enabling Graal JS.

For detailed information on this, see Graal VM JavaScript.

Default Email Sender

Platform 6.2 now provides an option to set a display name along with default email sender applicable for SMTP server only. This is helpful for the Platform users who could only override the default email settings at tenant level while changing the default email sender. This enhancement eliminates mismatches between the email address provided by the user and the display name value which is taken from current user's name.

For detailed information on this, see Account administration and security settings.

Performance Improvements

This release includes optimizations on performance in several areas towards our continued focus on Platform scalability and stability. Some of the key highlighted areas are:

  1. Data Import Performance: Data import performance has been improved significantly for both Bulk and Normal modes. The bulk mode has a higher level of performance improvement as compared to the Normal mode. These improvements are much higher when the SkipUniqueCheck option is set to True.

  2. Data Export Performance: Additionally, there has been a substantial improvement in the performance of list view data export with different file formats like .xlsx and .csv. The .xls and .xlsx file formats have shown higher level of performance improvement as compared to other export file formats.

  3. REST API 1.0 Performance: Platform 6.2 bags certain performance optimizations for some key Data APIs from REST API 1.0 as listed below.

    1. getPage

    2. getDataField

    3. getRecord

    4. getRelationships

    5. getCount

  4. Search Indexing Jobs Performance: This release pairs performance optimizations and enhanced search indexing, job queuing, and processing. Because job queuing is now configurable, the batch size has been augmented and set up for swifter data management.

  5. UI Performance Improvements: The performance of UI CRUD operations has been measurably optimized. The better performance is especially noticeable for records with thousands of related records which take a considerable amount of time to edit and save. Additionally, there is a significant improvement in performance while utilizing charts.
